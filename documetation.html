RBAC Blog Application - Documentation

1. Backend

Tech Stack: Node.js, Express.js, JWT Authentication

Features:

  • User Authentication using JWT (JSON Web Tokens).
  • Role Assignment (e.g., admin, user).
  • Role-Based Access Control (RBAC) Middleware:
    • Only admins can create, update, or delete blog posts.
    • All authenticated users can read blog posts.

2. Frontend

Tech Stack: React.js, Axios (for API calls)

Features:

  • Authentication Pages:
    • Signup Form
    • Login Form
  • User Dashboard:
    • View list of blog posts.
    • View single blog post details.
  • Admin Dashboard:
    • Create new blog posts.
    • Edit existing blog posts.
    • Delete blog posts.

3. Database

Options: MongoDB or PostgreSQL

Data Models:

  • User Model:
    • name: String
    • email: String (unique)
    • role: String ("user" or "admin")
    • password: Hashed String
  • Blog Post Model:
    • title: String
    • content: String
    • author: User Reference
    • timestamp: Date

4. Optional Feature

  • Email Verification during user signup using a verification token and email link.

5. Application Flow

Authentication:

  1. User signs up with name, email, and password.
  2. Backend hashes the password and stores user info with default role (user).
  3. On login, backend verifies credentials and issues a JWT token.

Authorization:

  1. Each API request includes the JWT token.
  2. Middleware checks user role.
  3. Allow or deny access based on role (e.g., only admins can create posts).

Frontend Interaction:

  1. User accesses the login/signup page.
  2. Upon login, token is stored (localStorage or cookie).
  3. Users can view posts.
  4. Admins see additional options (create, edit, delete posts).
